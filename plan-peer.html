Section 01

Technology Decision — Why Nostr

Requirements: fully client-side, no backend, async (users don't need to be online simultaneously), pairing via shared hash, free, reliable. The key constraint that kills most candidates is async — WebRTC-based tools require both peers online at the same moment.

Nostr Protocol

✓ Chosen
  • 100+ independent relay operators
  • Events stored on relays → async by design
  • secp256k1 identity baked in
  • Relay redundancy: any 2 of 5 working = ✓
  • nostr-tools v2 — ~50KB, actively maintained
  • Millions of real users, battle-tested
  • Events are signed + verifiable
  • No account, no config, no cost

Gun.js

✗ Rejected
  • Public relays (gun.eco, wallie, heroku) unreliable / offline frequently
  • Relay mesh has a single community bottleneck
  • HAM CRDT works well in theory
  • SEA crypto is solid
  • Infrastructure too fragile for production

Trystero

✗ Skip
  • Elegant WebRTC signaling library
  • Can use Nostr/BitTorrent as signal
  • Both peers must be online
  • Data lost if recipient is offline
  • Great for live chat — wrong for log sync

Firebase / Supabase

✗ Skip
  • Reliable, realtime, offline support
  • Requires account + project setup
  • Data on vendor servers
  • Violates "no backend" principle
  • Lock-in risk, free tier limits

What is Nostr?

Nostr (Notes and Other Stuff Transmitted by Relays) is an open protocol for decentralized communication. Clients publish cryptographically signed events to any number of relays. Relays are simple WebSocket servers that store and serve events. Anyone can run a relay — there's no central authority. The protocol is NIP-01 standard; events are JSON blobs signed with secp256k1.

💡

The key insight: Nostr relays are essentially persistent message stores with filter-based subscriptions. User A publishes a workout event → relay stores it. User B subscribes to that room's events → gets it when they next come online. Neither user needs to be online simultaneously. This is fundamentally different from WebRTC.

Relay Network (what we connect to)

We connect to 5 relays simultaneously via SimplePool from nostr-tools. Events are published to all; subscriptions read from all (deduplicated by event ID). If 3 of 5 are down, the app still syncs.

wss://relay.damus.io
wss://relay.nostr.band
wss://nos.lol
wss://nostr.wine
wss://relay.snort.social

These relays are independently operated by large teams (Damus, Snort, nos.lol, Nostr.band) with strong uptime incentives. They're free to use for read/write. Contrast this with Gun's handful of community Heroku dynos that go to sleep.

📦

Dependency: npm install nostr-tools — that's the only new runtime dep. ~50KB gzipped. Provides: key generation, event signing, SimplePool for multi-relay connections, filter subscriptions, and NIP encodings. No other packages needed.

What "Room" Means on Nostr

We define a custom #r tag (NIP-12 generic tag query). All FitnessPal buddy events carry ["r", roomHash]. To subscribe to a crew, query: { kinds: [31337, 31338], "#r": [roomHash] }. The relay filters and returns only events in that room. Two different invite codes produce two different room hashes — crews are completely isolated.

🔑

Room isolation: roomHash = hex(SHA-256(inviteCode)).slice(0, 24). Anyone who doesn't know the original invite code cannot discover the room hash (preimage resistance). Relay sees an opaque hex string, not a room name or user identity.

Identity & Cryptographic Pairing

Identity Bootstrap

On first Buddy tab visit, nostr-tools generates a secp256k1 keypair. The public key becomes the user's permanent identifier. The private key signs all events — it never leaves the device.

Section 04

Data Model — Nostr Event Schema

We use two custom Nostr event kinds. Both are parameterized replaceable (30000–39999 range per NIP-01): re-publishing the same d tag value replaces the previous event, so re-syncing is idempotent.

// Kind 31337 — Workout Log entry (one per WorkoutLog.id)
{
  kind:       31337,
  pubkey:     "<user pubkey hex>",
  created_at: 1719273600,         // unix seconds
  tags: [
    ["d", "<WorkoutLog.id>"],     // dedup key → replaceable
    ["r", "<roomHash>"],         // room filter
    ["t", "fitnesspal"],         // app namespace
  ],
  content:    "<JSON.stringify(WorkoutLog)>",
  sig:        "<secp256k1 signature>"   // from finalizeEvent()
}

// Kind 31338 — Buddy Profile (one per user per room)
{
  kind:       31338,
  pubkey:     "<user pubkey hex>",
  created_at: 1719273600,
  tags: [
    ["d", "profile"],              // always "profile" → one record per pubkey
    ["r", "<roomHash>"],
    ["t", "fitnesspal"],
  ],
  content:    "<JSON.stringify(BuddyProfile)>",
  sig:        "<sig>"
}

BuddyProfile Shape

interface BuddyProfile {

  color:    string;   // OKLCH hue (deterministic from pubkey)

Section 05

Sync Strategy & Offline Emitter

Push / Pull Lifecycle

On Buddy tab mount (useBuddySync hook)
  ├─ 1. Get/create Nostr SimplePool singleton
  ├─ 2. For each BuddyRoom in fp.buddy.rooms:
  │     ├─ Compute lastFetch = fp.buddy.lastFetch[roomHash] ?? 0
  │     └─ pool.subscribeMany(RELAYS, [{
  │           kinds: [31337, 31338],
  │           "#r":  [roomHash],
  │           "#t":  ["fitnesspal"],
  │           since: lastFetch - 60,   // 60s overlap for clock skew
  │         }], { onevent: handleIncoming })
  └─ 3. Push own profile + last 30d logs to all relays

handleIncoming(event)
  ├─ verifyEvent(event)             // signature check — reject fakes
  ├─ kind 31338 → upsert buddyProfiles[pubkey]
  └─ kind 31337 → upsert buddyLogs[pubkey][logId]

Sync Emitter (useSyncEmitter hook)
  ├─ setInterval: push own profile + logs every SYNC_INTERVAL (5 min)
  ├─ document visibilitychange → "visible": re-push immediately
  └─ useHistory.log() fires: push the new log immediately

On push: update fp.buddy.lastFetch[roomHash] = Math.floor(Date.now()/1000)

Offline Behavior

Nostr is inherently async. When a user is offline:

  • They can still log workouts locally (existing behavior, unchanged).
  • On next app open with network: subscribe with since: lastFetchAt → relay returns all missed events.
  • Their own unpublished logs queue is pushed on reconnect.
  • No Gun-style offline queue to manage — Nostr relays store events for weeks/months.
⏱

No background sync needed: Service Workers in PWAs can't keep WebSocket connections alive reliably. Instead, sync happens on every app foreground. The since filter on Nostr means we only download what changed — subscriptions are cheap even after days offline.

Event Deduplication

Nostr events have a deterministic id (SHA-256 of serialized content). The same workout log published twice produces the same event ID — relays and our in-memory state both deduplicate by ID automatically. Re-syncing is always safe.

Key Code Sketch — buddy-publish.ts

import { finalizeEvent } from "nostr-tools/pure";
import { RELAYS, getPool } from "./nostr";

export async function publishWorkoutLog(
  log: WorkoutLog, roomHash: string, identity: BuddyIdentity
) {
  const event = finalizeEvent({
    kind:       31337,
    created_at: Math.floor(Date.now() / 1000),
    tags:       [["d", log.id], ["r", roomHash], ["t", "fitnesspal"]],
    content:    JSON.stringify(log),
  }, hexToBytes(identity.privkey));
  await Promise.allSettled(getPool().publish(RELAYS, event));
}

import { createStore, isObj, str, arr } from "./storage";
import type { BuddyIdentity, BuddyRoom } from "@/types";

const validateIdentity = (raw: unknown): BuddyIdentity | null => {
  if (!isObj(raw)) return null;
  return {
    privkey:  str(raw.privkey),
    pubkey:   str(raw.pubkey),
    alias:    str(raw.alias),
    initials: str(raw.initials),
    color:    str(raw.color),
  };
};

export const identityStore = createStore<BuddyIdentity | null>(
  "fp.buddy.identity", null, validateIdentity
);

const isRoom = (v: unknown): v is BuddyRoom => {
  if (!isObj(v)) return false;
  return typeof v.id === "string" && typeof v.code === "string";
};

export const roomsStore = createStore<BuddyRoom[]>(
  "fp.buddy.rooms", [], (raw) => arr<BuddyRoom>(raw, isRoom)
);

export const lastFetchStore = createStore<Record<string, number>>(
  "fp.buddy.lastFetch", {}, (raw) => {
    if (!isObj(raw)) return {};
    const res: Record<string, number> = {};
    for (const [k, v] of Object.entries(raw)) {
      if (typeof v === "number") res[k] = v;
    }
    return res;
  }
);

Key Code Sketch — buddy-subscribe.ts

import { verifyEvent } from "nostr-tools/pure";
import type { Event } from "nostr-tools/pure";
import type { WorkoutLog, BuddyProfile } from "@/types";

export function handleIncomingEvent(
  event: Event,
  onLog: (pubkey: string, log: WorkoutLog) => void,
  onProfile: (pubkey: string, profile: BuddyProfile) => void
) {
  if (!verifyEvent(event)) {
    console.warn("Invalid signature:", event.id);
    return;
  }

  try {
    if (event.kind === 31337) {
      const log = JSON.parse(event.content) as WorkoutLog;
      if (log && typeof log.id === "string") {
        onLog(event.pubkey, log);
      }
    } else if (event.kind === 31338) {
      const profile = JSON.parse(event.content) as BuddyProfile;
      if (profile && typeof profile.alias === "string") {
        onProfile(event.pubkey, profile);
      }
    }
  } catch (err) {
    console.error("Failed to parse event:", err);
  }
}

Key Code Sketch — buddy-metrics.ts

import type { WorkoutLog } from "@/types";

export function getWeeklyCount(logs: WorkoutLog[]): number {
  const now = new Date();
  const startOfWeek = new Date(now.setDate(now.getDate() - now.getDay() + 1));
  startOfWeek.setHours(0,0,0,0);
  return logs.filter(l => l.status === "completed" && new Date(l.date) >= startOfWeek).length;
}

export function getStreak(logs: WorkoutLog[]): number {
  const dates = Array.from(new Set(
    logs.filter(l => l.status === "completed")
        .map(l => new Date(l.date).toDateString())
  )).map(d => new Date(d)).sort((a,b) => b.getTime() - a.getTime());

  if (dates.length === 0) return 0;
  const today = new Date(); today.setHours(0,0,0,0);
  if ((today.getTime() - dates[0].getTime()) / 86400000 > 1) return 0;

  let streak = 1;
  for (let i = 0; i < dates.length - 1; i++) {
    const diff = (dates[i].getTime() - dates[i+1].getTime()) / 86400000;
    if (diff <= 1.1) streak++; else break;
  }
  return streak;
}

export function getConsistencyPct(logs: WorkoutLog[]): number {
  const uniqueDays = new Set(logs.filter(l => l.status === "completed").map(l => new Date(l.date).toDateString()));
  return Math.min(100, Math.round((uniqueDays.size / 30) * 100));
}

  const [user] = useStore(userStore);

  useEffect(() => {
    if (identity) return;
    const secret = generateSecretKey();
    const privkey = Array.from(secret).map(b => b.toString(16).padStart(2, "0")).join("");
    const pubkey = getPublicKey(secret);
    
    const alias = user.name || "Athlete";
    const initials = alias.split(" ").map(n => n[0]).join("").slice(0, 2).toUpperCase();
    
    let hash = 0;
    for (let i = 0; i < pubkey.length; i++) hash = pubkey.charCodeAt(i) + ((hash << 5) - hash);
    const color = `${Math.abs(hash % 360)}`;

    setIdentity({ privkey, pubkey, alias, initials, color });
  }, [identity, user.name, setIdentity]);

  return identity;
}

import { useCallback } from "react";
import { useStore } from "./useStore";
import { roomsStore } from "@/lib/buddy-store";

export async function hashRoomCode(code: string): Promise<string> {
  const msgBuffer = new TextEncoder().encode(code.toUpperCase().trim());
  const hashBuffer = await crypto.subtle.digest("SHA-256", msgBuffer);
  return Array.from(new Uint8Array(hashBuffer))
              .map(b => b.toString(16).padStart(2, "0")).join("").slice(0, 24);
}

export function useBuddyRooms() {
  const [rooms, setRooms] = useStore(roomsStore);

  const createRoom = useCallback(async (label: string) => {
    const array = new Uint8Array(4);
    crypto.getRandomValues(array);
    const code = Array.from(array).map(b => b.toString(36).padStart(2, "0")).join("").slice(0, 8).toUpperCase();
    const id = await hashRoomCode(code);
    const room: BuddyRoom = { id, code, label: label.trim() || "Gym Crew", joinedAt: new Date().toISOString(), isCreator: true };
    setRooms(prev => [...prev.filter(r => r.id !== id), room]);
    return room;
  }, [setRooms]);

  const joinRoom = useCallback(async (code: string, label?: string) => {
    const id = await hashRoomCode(code);
    const room: BuddyRoom = { id, code: code.toUpperCase().trim(), label: label?.trim() || "Gym Crew", joinedAt: new Date().toISOString(), isCreator: false };
    setRooms(prev => [...prev.filter(r => r.id !== id), room]);
    return room;
  }, [setRooms]);

  const leaveRoom = useCallback((id: string) => {
    setRooms(prev => prev.filter(r => r.id !== id));
  }, [setRooms]);

  return { rooms, createRoom, joinRoom, leaveRoom };
}

Key Code Sketch — useBuddySync.ts

import { useEffect, useState } from "react";
import { getPool, RELAYS } from "@/lib/nostr";
import { handleIncomingEvent } from "@/lib/buddy-subscribe";
import { useStore } from "./useStore";
import { roomsStore, lastFetchStore } from "@/lib/buddy-store";
import type { BuddyProfile, WorkoutLog } from "@/types";

export function useBuddySync() {
  const [rooms] = useStore(roomsStore);
  const [lastFetch, setLastFetch] = useStore(lastFetchStore);
  const [profiles, setProfiles] = useState<Record<string, BuddyProfile>>({});
  const [logs, setLogs] = useState<Record<string, WorkoutLog[]>>({});
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    if (rooms.length === 0) { setLoading(false); return; }

    const roomIds = rooms.map(r => r.id);
    const minSince = Math.min(...roomIds.map(id => lastFetch[id] || 0));
    const since = Math.max(0, minSince - 60);

    const sub = getPool().subscribeMany(RELAYS, [{
      kinds: [31337, 31338],
      "#r": roomIds,
      since
    }], {
      onevent(event) {
        handleIncomingEvent(event,
          (pubkey, log) => setLogs(prev => {
            const userLogs = prev[pubkey] || [];
            if (userLogs.some(l => l.id === log.id)) return prev;
            return { ...prev, [pubkey]: [...userLogs, log].sort((a,b) => b.date.localeCompare(a.date)) };
          }),
          (pubkey, profile) => setProfiles(prev => {
            const ext = prev[pubkey];
            if (ext && ext.lastSeen >= profile.lastSeen) return prev;
            return { ...prev, [pubkey]: profile };
          })
        );
      },
      oneose() {
        setLoading(false);
        const now = Math.floor(Date.now() / 1000);
        setLastFetch(prev => {
          const next = { ...prev };
          roomIds.forEach(id => next[id] = now);
          return next;
        });
      }
    });

    return () => sub.close();
  }, [rooms, lastFetch, setLastFetch]);

  return { profiles, logs, loading };
}

Key Code Sketch — useSyncEmitter.ts

import { useCallback, useEffect } from "react";
import { finalizeEvent } from "nostr-tools/pure";
import { getPool, RELAYS } from "@/lib/nostr";
import { useBuddyIdentity } from "./useBuddyIdentity";
import { useStore } from "./useStore";
import { roomsStore } from "@/lib/buddy-store";
import { historyStore } from "@/lib/stores";
import type { BuddyProfile } from "@/types";

export function useSyncEmitter() {
  const identity = useBuddyIdentity();
  const [rooms] = useStore(roomsStore);
  const [history] = useStore(historyStore);

  const pushSync = useCallback(async () => {
    if (!identity || rooms.length === 0) return;

    const hexToBytes = (hex: string) => {
      const bytes = new Uint8Array(hex.length / 2);
      for (let i = 0; i < bytes.length; i++) bytes[i] = parseInt(hex.substring(i*2, i*2+2), 16);
      return bytes;
    };
    const privateKeyBytes = hexToBytes(identity.privkey);
    const nowSec = Math.floor(Date.now() / 1000);
    const profile: BuddyProfile = { alias: identity.alias, initials: identity.initials, color: identity.color, lastSeen: new Date().toISOString() };
    
    const thirtyDaysAgo = new Date(); thirtyDaysAgo.setDate(thirtyDaysAgo.getDate() - 30);
    const pushLogs = history.filter(l => l.status === "completed" && new Date(l.date) >= thirtyDaysAgo);

    for (const room of rooms) {
      const profEv = finalizeEvent({
        kind:       31338,
        created_at: nowSec,
        tags:       [["d", "profile"], ["r", room.id], ["t", "fitnesspal"]],
        content:    JSON.stringify(profile),
      }, privateKeyBytes);
      getPool().publish(RELAYS, profEv);

      for (const log of pushLogs) {
        const logEv = finalizeEvent({
          kind:       31337,
          created_at: Math.floor(new Date(log.date).getTime() / 1000),
          tags:       [["d", log.id], ["r", room.id], ["t", "fitnesspal"]],
          content:    JSON.stringify(log),
        }, privateKeyBytes);
        getPool().publish(RELAYS, logEv);
      }
    }
  }, [identity, rooms, history]);

  useEffect(() => {
    pushSync();
    const interval = setInterval(pushSync, 300000);
    const onVisible = () => document.visibilityState === "visible" && pushSync();
    document.addEventListener("visibilitychange", onVisible);
    return () => { clearInterval(interval); document.removeEventListener("visibilitychange", onVisible); };
  }, [pushSync]);

  return pushSync;
}

Section 10

Security & Privacy

RiskMitigationRating
Stranger discovers room by guessing invite code 8-char base36 = ~2.8 trillion combinations. Brute-force via relay would require millions of subscriptions — easily detectable / rate-limited by any reasonable relay operator. If joined: sees alias, workout dates, muscle groups, style. No sensitive data. Low risk
Event tampering / impersonation verifyEvent() from nostr-tools checks the secp256k1 signature on every incoming event. Events with invalid sigs are silently dropped before reaching app state. A relay cannot forge events it relays. Safe
Private key exposure privkey is in fp.buddy.identity (localStorage). It's read only in buddy-publish.ts to call finalizeEvent() — never logged, never included in published content, never sent to a relay directly. Safe
API key leakage via Nostr OpenRouter API key lives in fp.ai. Buddy sync code (buddy-publish.ts, buddy-subscribe.ts) never imports from fp.ai. Enforced by SRP module boundaries. Safe
XSS via buddy-supplied content All buddy-supplied strings (alias, initials) are rendered as text nodes, never as innerHTML. Alias capped at 32 chars; initials capped at 2 chars. WorkoutLog fields are typed enums — not free text — so injection surface is minimal. Mitigated
Relay data permanence Nostr relays store events, but most major relays prune old events after 30–90 days. Parameterized replaceable events (kind 31337/31338) that aren't re-published expire naturally. Users can "delete" by publishing a NIP-09 deletion event (optional enhancement). Acceptable
Relay outage 5 relays configured; Promise.allSettled() on publish means partial success is fine. Subscription via SimplePool.subscribeMany reads from all — whichever relays respond. App remains fully functional offline; sync catches up on reconnect. Resilient
⚠️

Disclosure to users: Workout metadata (dates, muscle groups, style, duration) published to Nostr relays is accessible to anyone who knows the room hash. The invite code is the only access barrier. Show a brief "Your crew data is public to anyone with this code" notice when creating the first room. One sentence, not a modal wall.

6 Should we use Nostr kind 0 (standard profile) instead of our custom kind 31338? No — kind 0 is a global Nostr identity. We don't want to write to users' global Nostr profiles. Room-scoped kind 31338 keeps FitnessPal data self-contained.
